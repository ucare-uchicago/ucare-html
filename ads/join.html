    Reading List

    [Only do this if Haryadi tells you so. Otherwise you can skip for now.]

  • Please visit UCARE reading list (http://ucare.cs.uchicago.edu/readings. Please read the README file first. Haryadi will tell you what reading is necessary for your project.

    Here's a required reading list depending on your research project. If you work on project related to:

    • Cloud computing, then read s-cloudos.html (Data-Center Stack section).
      (Minimally, you should understand the basic architecture/functionalities of GoogleFS/HDFS, MapReduce/Hadoop, BigTable/HBase, Dynamo/Cassandra)
    • File/storage systems, then read s-cloudos.html (File System 101 section).
    • SSD, then read s-ssd.html (SSD 101).
  • (Note that if you feel you don't have a strong systems background, then you need to self-learn Operating System concepts. Please go through the materials in OSTEP free book (every chapter). This is a very short direct-to-the-point book, so you should be able to absorb things quickly. You don't have to memorize everything. As long as you understand the concepts, it's good enough -- in the future if you need to go deep, you know which chapter to look for. When you reach the persistence chapters, make sure you digest things slowly).

  • Finally, make sure you read UCARE publications related to your project. Our publications can be found here: http://ucare.cs.uchicago.edu/pubs.html. If you have detailed questions about some of the papers, please talk directly to the student authors.
